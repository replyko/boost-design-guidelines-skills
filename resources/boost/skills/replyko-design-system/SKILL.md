# Replyko Design System

Use this skill when working on shared Replyko visual direction.

## When To Use This Skill

Activate this skill when the task involves:
- colors or design tokens
- typography
- button hierarchy
- logo rules
- CTA styling
- trust badges or verification styling
- dark section rules
- aligning the marketing site and the app

## Non-Negotiables

- Primary Brand: `#4F46E5`
- Primary Dark: `#3730A3`
- Primary Soft: `#EEF2FF`
- CTA Accent: `#FF512F`
- Success / trust: `#10B981`
- Neutrals: Tailwind Slate only
- Headings use `Space Grotesk`
- Body and UI use `Manrope`

## Button Hierarchy

1. Coral CTA for primary conversion
2. Indigo filled for important secondary actions
3. Ghost or outline for tertiary actions

## Cross-Repo Rule

Keep one Replyko identity across both repos.

Marketing can be more spacious and expressive.

The app can be denser and more utility-focused.

Do not solve repo differences by inventing separate brand systems.

## App Migration Rule

The app currently contains teal-based tokens in local code.

Do not expand teal as a long-term design direction when the task can move shared UI toward Replyko Indigo and Coral.
