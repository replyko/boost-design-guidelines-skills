# Replyko Main Site Rules

Add this to the existing Laravel Boost `CLAUDE.md` in `replyko.com`. It is a project-specific add-on, not a replacement for the framework rules.

This repo is `replyko.com`, the marketing website for Replyko.

Follow these rules on every task:

## Product Context

- This repo exists to explain the product, build trust, and drive signups
- Optimize for clarity, conversion, and trust
- Prefer marketing-site structure over dashboard-like density

## Shared Design System

### Colors
- Primary Brand: `#4F46E5`
- Primary Dark: `#3730A3`
- Primary Soft: `#EEF2FF`
- CTA Accent: `#FF512F` and only for conversion CTAs
- Neutrals: Tailwind Slate only
- Success: `#10B981`

### Typography
- Headings: `Space Grotesk`
- Body/UI: `Manrope`
- Load from `fonts.bunny.net`

### 60-30-10 Rule
- 60% neutral slate and white
- 30% Indigo brand usage
- 10% Coral CTA usage only

### Button Hierarchy
1. Coral CTA for main conversion
2. Indigo filled for important secondary actions
3. Ghost or outline for tertiary actions

### Brand Guardrails
- Never recolor the Replyko mark
- Never use Coral as decoration
- Never use Gray or Zinc neutrals
- Meta API and trust badges must use Emerald

## Repo-Specific Guidance

- Keep hero sections immediately understandable
- Use Coral on primary conversion buttons like `Get Started Free`
- Keep layouts spacious and high signal
- Use Indigo and Primary Soft to frame sections and content groups
- Use dark sections sparingly and keep Coral CTAs unchanged inside them
- Follow existing Blade, route, and content-model patterns used by blog, competitor, use-case, and legal pages
